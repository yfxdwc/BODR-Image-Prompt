# BODR Image Prompt v0.3.0-alpha

**Release theme:** Multilingual provenance-aware prompt vault

This alpha upgrades BODR Image Prompt from a small public demo into a richer read-only prompt vault. The GitHub Pages preview now combines multiple attribution-preserving sample sources, provides English / Traditional Chinese / Simplified Chinese prompt variants for every public sample item, and tracks which prompt is the source/original versus derived translations or script conversions.

Local add/edit/private library management remains local-install only.

## Demo links

- Version chooser: <https://eddietyp.github.io/BODR-Image-Prompt/>
- Current 0.3 preview: <https://eddietyp.github.io/BODR-Image-Prompt/v0.3/>
- Archived 0.2 preview: <https://eddietyp.github.io/BODR-Image-Prompt/v0.2/>
- Archived 0.1 alpha demo: <https://eddietyp.github.io/BODR-Image-Prompt/v0.1/>

The GitHub Pages demos are read-only online sandboxes using public sample prompts and compressed preview images. Run the app locally to create and manage your own private full library.

## Highlights

- Expanded the public read-only vault to **510 references** from two attribution-preserving sample sources:
  - `wuyoscar/gpt_image_2_skill` / CC BY 4.0
  - `freestylefly/awesome-gpt-image-2` / MIT
- Upgraded sample manifests to **schema v2** with explicit prompt provenance:
  - exactly one source/original prompt per item
  - machine translations and OpenCC script conversions are marked as derived
  - Origin is prompt metadata, not a separate duplicate prompt block
- Added full multilingual prompt coverage for every public sample item:
  - English
  - Traditional Chinese
  - Simplified Chinese
- Revamped prompt viewing/copying flows:
  - Origin / English / zh-Hant / zh-Hans prompt options
  - UI-language-aware labels such as `Origin`, `原文`, `英文`, `繁中`, `簡中`
  - source/original prompt tabs receive dedicated styling and badges
- Added localized collection metadata for the public sample vault.
- Shortened reference/source labels while preserving full source URLs through the existing URL button.
- Added stricter backend validation for source/original prompt markers.
- Added a reusable helper for maintaining derived prompt translations: `backend/services/fill_sample_manifest_translations.py`.
- Refreshed README, attribution, release notes, and public demo wording around the local-first architecture:
  - GitHub Pages is a static read-only vault
  - add/edit/private library management requires local installation

## Local setup

See the README for setup details. Short version:

```bash
./scripts/setup.sh
./scripts/start.sh
```

Optional sample library:

```bash
./scripts/install-sample-data.sh en
./scripts/install-sample-data.sh zh_hant awesome-gpt-image-2
```

Requires Python 3.10+ and Node.js 24 for the current verified development path.
