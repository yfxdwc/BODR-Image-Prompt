# BODR Image Prompt v0.2.0-alpha

**Release theme:** current 0.2 preview

This alpha focuses on making the read-only sandbox and local app more useful as a public prompt vault, with stronger mobile browsing, versioned Pages previews, and attribution-preserving sample data.

## Demo links

- Version chooser: <https://eddietyp.github.io/BODR-Image-Prompt/>
- Current 0.2 preview: <https://eddietyp.github.io/BODR-Image-Prompt/v0.2/>
- Archived 0.1 alpha demo: <https://eddietyp.github.io/BODR-Image-Prompt/v0.1/>

The GitHub Pages demos are read-only online sandboxes using public sample prompts and compressed preview images. Run the app locally to create and manage your own private full library.

## What changed since v0.1

- Mobile users now default to **Cards** view when there is no saved view preference.
- Mobile Cards use a stable two-column masonry layout instead of CSS multi-column rendering.
- Card display is image-first with a title-only overlay.
- Mobile card actions remain touch-visible for prompt copy, save/favorite, and edit where mutations are enabled locally.
- Long, wide, and tall images display adaptively using preview/original images before cropped thumbnails.
- Detail modal is more phone-first: image above prompt/content, floating close/favorite/edit controls, and natural image sizing.
- Mobile header is compact: Filter, centered brand, Config, search, count, and Explore/Cards switch.
- Selected collections on mobile use a bottom selected-collection dock with adaptive text sizing, while the Filter button shows only a purple selected state.
- The duplicate Explore focus panel was removed; focus remains in the constellation map.
- Count copy is shorter: `references` / `個參考`.

## Versioned GitHub Pages

GitHub Pages now publishes multiple preview paths:

- `/v0.1/` preserves the original alpha sandbox as an archive.
- `/v0.2/` publishes the current 0.2 prompt-vault preview.
- `/` is a lightweight version chooser linking to both versions.

## Local-first architecture reminder

The app remains local-first: SQLite metadata and image files stay on your own machine when self-hosted locally. The public GitHub Pages demo is static/read-only and does not include private runtime databases, originals, `.env` files, backups, or local work directories.

## Sample data and attribution

The public sample library uses attribution-preserving sample material from `wuyoscar/gpt_image_2_skill` / CC BY 4.0 and `freestylefly/awesome-gpt-image-2` / MIT. The sample images are compressed for web preview in the online sandbox, and prompt provenance marks the source/original prompt where available.

## Known limitations

- This remains an alpha release.
- The online sandbox is read-only; use local setup for add/edit/delete/private libraries.
- Mobile Explore contained pan/pinch polish is planned for a later pass.
- Public sample data is for demonstration and attribution-preserving evaluation, not a replacement for your private prompt library.

## Local setup

See the README for setup details. Short version:

```bash
./scripts/setup.sh
./scripts/start.sh
```

Optional sample library:

```bash
./scripts/install-sample-data.sh en
./scripts/install-sample-data.sh zh_hant awesome-gpt-image-2
```

Requires Python 3.10+ and Node.js 24 for the current verified development path.
