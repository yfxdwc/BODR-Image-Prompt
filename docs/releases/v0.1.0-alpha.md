# BODR Image Prompt v0.1.0-alpha

This alpha release packages the first public-ready version of **BODR Image Prompt**: a local-first web app for saving generated images together with the prompts behind them.

## Try the sandbox

Read-only online sandbox: <https://eddietyp.github.io/BODR-Image-Prompt/>

The read-only online sandbox uses public sample prompts and compressed WebP preview images so visitors can try the browse, search, detail, and copy-prompt flow without installing anything. It is not a hosted cloud library; run the app locally to create and manage a private full library.

## What is included

- Local-first app architecture using FastAPI, SQLite, local image files, and a Vite/React frontend.
- Visual Explore view for browsing prompt references by collection.
- Masonry Cards view for dense prompt/image scanning.
- Search across titles, prompt text, tags, collections, source metadata, and notes.
- Detail modal with prompt language tabs and one-click prompt copy.
- Add/edit/delete/archive flows for local prompt references.
- Optional sample library installer.
- GitHub Pages read-only demo build using static JSON and compressed sample media.
- CI workflow for Python tests, local frontend build, and demo build.

## Install locally

Requirements:

- Python 3.10+
- Node.js 20+ recommended
- npm

Quick start:

```bash
git clone https://github.com/EddieTYP/BODR-Image-Prompt.git
cd BODR-Image-Prompt
./scripts/setup.sh
./scripts/start.sh
```

Open <http://127.0.0.1:8000/>.

Optional sample library:

```bash
./scripts/install-sample-data.sh en
```

## Known limitations

- This is an alpha release; expect rough edges in UI polish and packaging.
- macOS and Linux are the primary local-install targets today; Windows users should use WSL 2 for the current shell-script workflow.
- The online sandbox is read-only and uses compressed preview images.
- The app does not provide hosted cloud sync or accounts; local data remains on the user's machine.
- Native Windows PowerShell scripts, Docker Compose, richer backup/import UI, and optional semantic search are future work.
