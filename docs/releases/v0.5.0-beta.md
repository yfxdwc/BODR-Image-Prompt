# BODR Image Prompt v0.5.0-beta

**Release theme:** Local Generation Studio

This beta keeps GitHub Pages as an **Online Read Only Demo** while making local installs feel like a practical generation workbench: connected users can generate from a new prompt or existing reference, pick ChatGPT-style aspect ratio and quality settings, let jobs start automatically, review outputs in a focused workbench, and save results into their private prompt library only when ready.

The online demo remains read-only. Add/edit/private library management and direct generation require a local install.

## Highlights

- Promoted the local generation flow into a two-column **Generation Studio** on desktop and a stacked card flow on mobile.
- Added ChatGPT-style aspect ratio controls for local generation:
  - `1:1 Square`
  - `3:4 Portrait`
  - `9:16 Vertical`
  - `4:3 Landscape`
  - `16:9 Wide`
- Preserved aspect ratio provenance by storing the requested ratio separately from the source prompt and composing provider-specific prompt instructions at generation time.
- Added a simpler quality selector with `Auto`, `Standard`, and `High` options.
- Changed local generation so `Generate` creates and starts native-provider jobs automatically; the old manual `Run` path remains only for retry/debug compatibility.
- Added an in-process native generation queue with a default limit of two concurrent jobs.
- Added `Cancel` for queued and running generation jobs.
- Added an explicit `cancelled` status so cancelled jobs are distinct from discarded results.
- Added workbench auto-refresh while visible jobs are queued or running.
- Made generation queue rows open the relevant workbench and focus the selected job.
- Made variant-generation notifications open the variant workbench first instead of jumping straight back to the source image.
- Added focused-job highlighting so users can see which generation job was opened from the queue.
- Tightened the desktop generation layout, centered the connected provider pill, and aligned the global Add/Generate buttons.
- Made Cards view the default on desktop and mobile when no saved preference exists; saved Explore/Cards preferences are still respected.
- Kept ChatGPT OAuth generation provider setup optional and local-only through `openai_codex_oauth_native`.
- Fresh ChatGPT OAuth onboarding QA has been checked for this release cycle.

## Install

Install the v0.5 beta from GitHub Release assets:

```bash
curl -fsSL https://raw.githubusercontent.com/EddieTYP/BODR-Image-Prompt/main/scripts/install.sh | bash -s -- --version v0.5.0-beta
BODR-Image-Prompt start
```

Update an existing install to v0.5 beta:

```bash
BODR-Image-Prompt update --version v0.5.0-beta
```

Rollback to the previous installed version:

```bash
BODR-Image-Prompt rollback
```

## Known limitations

- This remains a beta prerelease.
- The Online Read Only Demo is static and read-only; it does not expose Add/Edit/Generate controls.
- ChatGPT OAuth direct generation is local-only and experimental because it depends on the ChatGPT/Codex backend path.
- Running-job cancellation is soft cancellation: the provider request may still finish, but the app ignores/rejects the result if the job has already been cancelled.
- The native generation queue is currently in-process; future releases may add stronger startup recovery and persistence behavior.
- No interactive release picker/list is implemented yet; use `--version <tag>` for selected releases.
