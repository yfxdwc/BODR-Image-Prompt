# BODR Image Prompt v0.6.2-beta

**Release theme:** Update Reliability Fixes

This beta patch fixes updater reliability for installed local apps, especially macOS launchd services where the browser-triggered updater can run with a restricted system environment.

The online demo remains read-only. Add/edit/generation/private library management require a local install.

## Highlights

- Fixed browser-triggered app updates under macOS launchd so the installer reuses the running app runtime Python instead of falling back to the system Python.
- Fixed CLI `BODR-Image-Prompt update` to pass the installed app runtime Python through to the installer.
- Improved launchd service detection and scheduled restart handling for installations that use a non-default service label.
- Kept the v0.6.1 Save-as-new metadata polish, direct Download actions, and original-image fullscreen behavior.
- No database schema change and no data migration required.

## Install

Install the v0.6.2 beta from GitHub Release assets:

```bash
curl -fsSL https://raw.githubusercontent.com/EddieTYP/BODR-Image-Prompt/main/scripts/install.sh | bash -s -- --version v0.6.2-beta
BODR-Image-Prompt start
```

Update an existing install to v0.6.2 beta:

```bash
BODR-Image-Prompt update --version v0.6.2-beta
```

Rollback to the previous installed version:

```bash
BODR-Image-Prompt rollback
```

## Known limitations

- This remains a beta prerelease.
- The Online Read Only Demo is static and read-only; it does not expose Add/Edit/Generate controls.
- ChatGPT OAuth direct generation is local-only and experimental because it depends on the ChatGPT/Codex backend path.
- Running-job cancellation is soft cancellation: the provider request may still finish, but the app ignores/rejects the result if the job has already been cancelled.
- The native generation queue is currently in-process; future releases may add stronger startup recovery and persistence behavior.
- Account Management is planned for v0.7 and is not included in this release.
