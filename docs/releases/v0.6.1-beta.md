# BODR Image Prompt v0.6.1-beta

**Release theme:** Save-as-new Metadata & Image Actions

This beta patch refines the v0.6 local generation workflow after real use: Save-as-new metadata controls are easier to complete, image actions are clearer, and generated items start with less unwanted inherited text.

The online demo remains read-only. Add/edit/generation/private library management require a local install.

## Highlights

- Improved Save-as-new tag editing so comma-separated entry works naturally while still offering existing tag suggestions.
- Added smart Collection suggestions to the Save-as-new metadata grid, matching the Add/Edit modal workflow.
- Replaced the native Save prompt Source Language selector with compact original-language pills for `ENG`, `繁中`, and `簡中`.
- Defaulted notes for new generated items to empty text, avoiding unwanted generated note copy.
- Opened detail fullscreen images from the original image when available instead of the preview/thumbnail path.
- Added direct Download actions on cards, detail modals, and generated result previews.
- Kept GitHub Pages as an Online Read Only Demo while local installs retain add/edit/generation/private library management.

## Install

Install the v0.6.1 beta from GitHub Release assets:

```bash
curl -fsSL https://raw.githubusercontent.com/EddieTYP/BODR-Image-Prompt/main/scripts/install.sh | bash -s -- --version v0.6.1-beta
BODR-Image-Prompt start
```

Update an existing install to v0.6.1 beta:

```bash
BODR-Image-Prompt update --version v0.6.1-beta
```

Rollback to the previous installed version:

```bash
BODR-Image-Prompt rollback
```

## Known limitations

- This remains a beta prerelease.
- The Online Read Only Demo is static and read-only; it does not expose Add/Edit/Generate controls.
- ChatGPT OAuth direct generation is local-only and experimental because it depends on the ChatGPT/Codex backend path.
- Running-job cancellation is soft cancellation: the provider request may still finish, but the app ignores/rejects the result if the job has already been cancelled.
- The native generation queue is currently in-process; future releases may add stronger startup recovery and persistence behavior.
- Account Management is planned for v0.7 and is not included in this release.
